Системы контроля версий.

С помощью систем контроля версий возможо скачать самую последнюю версию проекта либо какую-то из предыдущих версий. Перед тем как начинать работу с проектом необходимо скачать последнюю версию, сделать изменения и затем сделать коммит (сохранить изменения) и пуш (отправить изменения в удалённый репозиторий).

Последовательность операций:

  1. получить (pull, update) — взять последнюю версию кода проекта.
  2. сделать правки в коде проекта.
  3. закомитить (commit) — утвердить изменения: написать комментарий к изменениям, выделить файлы которые меняются.
  4. отправить (push) — отправить изменения в репозиторий.

Работа с репозиторием

Когда репозиторий проекта уже существует, первое что нам надо узнать:

  1. Какой URL репозитория?
  2. Есть ли у моего пользователя доступ к этому репозиторию? И если есть, есть ли доступ на запись?

Первый pull (update)

Задание #1.1

  1. На диске или рабочем столе создать папку mobi.
  2. Скопировать URL репозитория:
    https://github.com/alekskorovin/mobi
  3. Открыть установленную ранее программу SourceTree.
  4. Нажать кнопку Клонировать / создать.
  5. Вставить скопированный ранее адрес репозитория в поле Исходный путь / URL:
  6. Заменить автоматически сгенерированный путь на путь к созданной ранее пустой папке mobi в поле Целевой путь:
  7. После указания правильного пути к пустой папке mobi нажать кнопку Клонировать:
  8. После клонирования репозитория в папку mobi нажать на название ветки master:
  9. Обратить внимание на последний коммит:
  10. Открыть папку mobi и убедиться что в ней появился файл index.html

После того как последняя версия кода проекта получена можно делать правки кода.

Делаем общую страницу

Задание #1.2

  1. По очереди делаются следующие шаги:
    • Первый разработчик находит div с классом со своим именем, например class="team-member-arina", и заполняет содержимое двух вложенных div-ов: <div class="team-member-name"></div>
    • Сохраняет файл.
    • Открывает SourceTree и нажимает на строчку Незакомиченные изменения:
    • Нажимает на чекбокс Файлы не в индексе:
    • Нажимает на кнопку Закоммитить
    • Добавляет осмысленный комментарий к коммиту и нажимает кнопку Закоммитить:
    • Нажимает кнопку Отправить чтобы отправить изменения на удалённый сервер GitHub:
  2. Следующий разработчик нажимает на кнопку Получить для получения последней версии с изменениями и проделывает все шаги из предыдущего пункта.

После того как все разработчики внесли изменения есть возможность опубликовать эту страницу в интернете.

Для этого последний кто делал правки делает следующее:

  • Нажать правой кнопкой на название ветки master и выбрать Переименовать master...
  • Ввести название ветки gh-pages и нажать ОК
  • Отправить изменения на сервер с помощью кнопки Отправить:
  • Теперь по адресу https://alekskorovin.github.io/mobi/index.html должна появиться опубликованная страница.

Создание собственного репозитория на GitHub

  1. Залогиниться на GitHub, зайти на главную страницу и нажать кнопку New repository в правой колонке:
  2. Ввести название своего репозитория и нажать кнопку Create repository:
  3. После переадресации на страницу репозитория, можно увидить URL репозитория https://github.com/alekskorovin/test-repo (в вашем случае будет другое имя) и использовать его для клонирования и дальнейшей работы как было описано в предыдущих пунктах.
  4. Надо создать пустую папку, склонировать туда репозиторий, т.к. он пустой то в папке ничего не появится.
  5. Добавить в папку файлы и закомитить их, затем отправить на сервер, после этого они появятся на GitHub-е.
